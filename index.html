All sole-developer projects, built and maintained at Ouitech Ltd.

Snagg — Field Engineer App
SOLE DEVELOPER · REACT NATIVE · EXPO (EAS) · REDUX
WEAK SIGNAL → FAST FAIL → OFFLINE

An offline-first job and task management app used daily by field engineers. Handles clock-in/out, signatures, photo reports, and dependencies entirely offline, with automatic background sync once connectivity returns. I found that in basements and plant rooms, weak — not dropped — signal caused sync requests to hang rather than fail, leaving the app looking frozen. I added request timeouts so a stalled sync now fails fast and drops cleanly into offline mode instead of leaving engineers stuck.

WeQuote Scanner App
SOLE DEVELOPER · EXPO DEV CLIENT · ANDROID
GOODS-IN → STOCK SYNCED

A barcode-driven inventory app for rugged Android handhelds. Engineers scan goods in and out of the warehouse, with stock levels updating in real time back to WeQuote.cloud. Built a resilient session-recovery flow (reset / lock / sign-out) for expired or mismatched device sessions, and administered the self-hosted Headwind MDM fleet behind it — kiosk-mode lockdown, silent OTA updates, and diagnosing device-side failures across the fleet.

Notification & Delivery Tracking System
SOLE DEVELOPER · AWS SES/SNS · PHP · POSTGRESQL
SILENT FAILURE → VISIBLE AS DATA

An event-driven pipeline — SES Configuration Sets → SNS → a PHP webhook handler → PostgreSQL — powering delivery, bounce, and complaint dashboards for end-users and admins. The point was simple: a failed email shouldn't be discovered from a support ticket. I also owned the email infrastructure end to end — domain verification, DKIM/SPF/DMARC, dedicated IP planning, and sandbox-to-production provisioning.
